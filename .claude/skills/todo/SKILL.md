# Todo Manager

## Neue Aufgabe

Wenn eine neue Aufgabe erstellt wird:

1. Analysieren
2. Priorität vergeben
3. Kategorie erkennen
4. Passende Skills auswählen
5. Große Aufgaben in Teilaufgaben zerlegen
6. In backlog.md eintragen

---

## Status

Aufgaben besitzen folgende Stati:

- Backlog
- Planung
- In Bearbeitung
- Review
- Erledigt
- Archiviert

---

## Skills

Ordne automatisch passende Skills zu.

Beispiele

Bug

→ bugfix

Neue Datenbank

→ database-change

Neues Modul

→ geraetehaus-patterns

Größeres Feature

→ planner

---

## Nach Abschluss

Nach jeder erledigten Aufgabe prüfen:

- neues Pattern?

→ EXAMPLES.md

- neue Erkenntnis?

→ LESSONS.md

- neue Projektregel?

→ CLAUDE.md
